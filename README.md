# Authentication Study Project

## About

This project was built as a learning exercise to understand how authentication works in web applications.

The goal was not to create a production-ready application, but to gain hands-on experience with backend fundamentals before starting a larger project (an ENEM study platform).

## Concepts Studied

During this project, I learned and practiced:

* REST API development with Express
* PostgreSQL integration using pg
* Environment variables with dotenv
* Password hashing with bcrypt
* JWT (JSON Web Tokens)
* User registration and login flows
* Protected routes
* Basic frontend-backend communication using fetch
* HTTP status codes and error handling

## Features Implemented

* User registration
* User login
* Password hashing before database storage
* JWT token generation
* JWT token verification
* Protected profile endpoint
* PostgreSQL database integration

## What I Learned

Some of the main lessons from this project were:

* Why passwords should never be stored in plain text
* How JWT authentication works
* The difference between authentication and authorization
* How a frontend communicates with a backend API
* How environment variables help protect sensitive data
* Basic project structure for backend applications

## Future Improvements

This project achieved its learning objectives, but several improvements could be made:

* Authentication middleware

## Purpose

This repository serves as a learning milestone and reference project while studying backend development.

The knowledge gained here will be applied to future projects, especially a larger educational platform focused on ENEM preparation.
